# End-to-End test results — Beauty.dev

## Automated test

`backend/tests/test_e2e.py::test_full_journey` walks the full master + client
journey in a single test against a real Postgres + Redis (via docker compose)
with stubbed Telegram and LLM.

```
docker compose -f docker-compose.yml run --rm -e ENVIRONMENT=test backend pytest tests/test_e2e.py -v
```

Result: **PASSED in 1.24s** (run on 2026-04-30).

Steps the test exercises:

| # | Step | Verifies |
|---|------|----------|
| 1 | `GET /api/me` first call | new master row, trial_ends_at set, bot_enabled=true |
| 2 | `PATCH /me` + services + schedule + seed manicure funnel | onboarding writes everything correctly |
| 3 | webhook `business_connection` | BusinessConnection row, master upserted |
| 4 | webhook `business_message` (client) ×2 | Conversation + Message rows; bot OUT replies generated by stubbed LLM |
| 5 | LLM returns `slot_intent` on 2nd call | Booking auto-created, reminders (24h skipped if <24h, 2h, feedback) scheduled |
| 6 | webhook `business_message` (master outgoing) | conversation flips to `human_takeover`, takeover_until set |
| 6b | another client message during takeover | bot stays silent (no new OUT row) |
| 7 | `POST /api/conversations/{id}/release` | state back to `bot`, takeover_until cleared |
| 8 | `deliver_due_reminders(now=+10d)` | 24h/2h/feedback rows dispatch idempotently, sent_at set |
| 9 | mark booking DONE + `recompute_for_master` | Client picks up `new` segment; surfaced via `GET /api/clients` |
| 10 | `generate_for_master` then `GET /api/insights` | placeholder/insight row written and served |

Plus dashboard sanity:
- `/api/analytics/dashboard` shows `bot_enabled=true` and `pending_takeovers=0`
  after release.

## Manual checklist (production)

After `docker compose -f docker-compose.prod.yml up -d --build` and webhook
registration on the live domain, run through:

1. **Mini App opens from BotFather** — open `@beauty_dev_bot` in Telegram, tap
   the Mini App button → onboarding renders with the master's `first_name`.
2. **Premium gate** — without Telegram Premium the Premium step shows the
   "Купить Premium" button; with Premium, it auto-advances.
3. **Onboarding completes** — name + niche → schedule → connect → Done.
   The Done step seeds a niche-appropriate funnel (`manicure` or
   `brows_lashes`).
4. **Business connection** — in Telegram → Settings → Telegram Business →
   Чат-боты → add `@beauty_dev_bot`. Within ~5s, the Connect screen pulses
   green; the API has a `business_connections` row with `is_enabled=true`.
5. **Client → bot reply** — write to the master's Telegram from another
   account. The bot replies in Russian within ~2s. Both messages appear in
   `/api/conversations/{id}` and the Mini App **Чаты** tab.
6. **Slot booking** — client confirms a time the bot proposed. A booking
   appears on the **Календарь** tab; the master gets a private push from
   `@beauty_dev_bot` ("Новая запись: <name>, ...").
7. **Reminders** — wait until ~2h before the booking; the client receives
   the 2h reminder. After the booking ends, ~2h later the feedback request
   arrives.
8. **Takeover** — in the master's Telegram, reply manually to the client.
   The Mini App **Чаты** tab shows that conversation as "вы". An incoming
   client message during the takeover window does NOT trigger a bot reply
   (verified in `messages` table).
9. **Release** — tap "Передать обратно боту" on the conversation card. The
   next client message gets a bot reply again.
10. **Kill switch** — flip `bot_enabled=false` in Settings. New client
    messages are recorded but the bot stays silent globally.
11. **Insights** — after a week of activity, `/api/insights` and the
    Аналитика screen surface a non-placeholder line (revenue trend, top
    service, etc.).
12. **Pricing + paywall** — wait out (or shorten) `trial_ends_at`. The
    Главная route returns the Paywall; tapping the CTA opens `/app/pricing`,
    where a Pro checkout opens the YooKassa confirmation_url. After
    `payment.succeeded` webhook, the master's `subscription_active_until`
    extends by 30 days and the Paywall is gone.
13. **Portfolio** — POST a few JPEGs to `/api/portfolio` from the Mini App.
    They're resized to long side 1280, served from
    `https://crm.plus-shop.ru/portfolio/<master_id>/<token>.jpg`. When the
    bot's `portfolio_request` flag fires, up to 3 random photos are sent.

## Known limitations / deferred items

- **AutoModes** and **BotSuggest** screens from the design are intentionally
  shipped as "Скоро" stubs (per DEV_PROMPT.md).
- **Pro+ team features** are placeholders — same Pro feature set, different
  price tier — until team members ship.
- **Recharts bundle weight** — frontend bundle is ~190 KB gzipped; consider
  dynamic-importing the analytics page if first-paint becomes a concern.
